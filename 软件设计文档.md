## 1. 背景与约束

工程师在本地 Linux 工作站上运行 Claude Code，需要用它操控远程 Linux 服务器——读写文件、执行命令、修改配置、运行编译任务。

**硬性约束**：

- 远程服务器只开放 SSH 访问，不安装任何额外软件
- 本地机器为 Linux，有 Python 3.8+ 和标准 SSH 工具链
- 通过 Claude Code 的 MCP 机制接入

---

## 2. 系统概述

remote-mcp 是一个运行在本地的 MCP server。它向 Claude Code 暴露一组工具（Read、Write、Edit、Bash、Glob、Grep），这些工具与 Claude Code 内置工具接口一致，但所有操作都在远程服务器上执行。Claude Code 通过 stdio MCP 协议调用工具，remote-mcp 将调用转发到远程服务器，只将结果传回本地。

```
┌─────────────────────────────────────────────────────┐
│                    本地机器                          │
│                                                     │
│  ┌──────────────┐   stdio MCP   ┌────────────────┐  │
│  │  Claude Code │ ◄───────────► │   remote-mcp   │  │
│  │  (MCP Client)│               │   MCP Server   │  │
│  └──────────────┘               └───────┬────────┘  │
│                                         │            │
│                                  SSHConnection       │
│                                  (paramiko)          │
└─────────────────────────────────────────────────────┘
                                          │ SSH
                    ┌─────────────────────┼──────────┐
                    │       远程服务器     │          │
                    │                     │          │
                    │         ┌───────────▼────────┐ │
                    │         │   bash (持久进程)  │ │
                    │         │   stdin/stdout 管道 │ │
                    │         └────────────────────┘ │
                    │                                 │
                    │         远程文件系统（原生访问） │
                    └─────────────────────────────────┘
```

> **Figure 1**: remote-mcp 运行在本地，持有到远程的 SSH 连接。文件操作通过 SFTP 完成，命令执行通过持久 bash 进程完成。Claude Code 无感知"远程"的存在。

**核心原则**：所有计算在远程发生，只有结果通过网络传输。例如 Grep 在远程执行后只返回匹配行，而不是把整个文件传到本地再搜索。

## 3. 设计决策

### 3.1 实现语言：Python + paramiko

使用 Python 实现，SSH 库选用 paramiko。

理由：paramiko 是纯 Python 的 SSH 实现，提供 Channel 级别的控制，可以创建持久 shell session 并精确管理其 stdin/stdout 管道——这是实现持久 Bash session 的基础。Anthropic 官方提供 Python MCP SDK，两者结合无阻抗。

### 3.2 MCP Transport：stdio

使用 stdio transport 接入 Claude Code。

理由：与 Claude Code 所有其他 MCP server 一致。进程生命周期由 Claude Code 管理，无需用户手动启动和维护，零网络端口配置。

### 3.3 每台主机一个 MCP server 实例

每个 remote-mcp 进程只连接一台远程主机，通过启动参数 `--host <name>` 指定。多台主机在 Claude Code 中注册多个 MCP server entry。

理由：单实例单连接，生命周期清晰，故障隔离简单。

### 3.4 工具接口与 Claude Code 内置工具一一对应

工具名称、参数名称、返回格式与 Claude Code 内置的 Read/Write/Edit/Bash/Glob/Grep 完全一致。

理由：Claude Code 对内置工具的调用策略已经过训练优化。接口一致时，Claude Code 会以相同策略调用远程版本，无需额外提示词适配。

### 3.5 文件操作通过 SFTP，命令执行通过持久 bash

文件操作（Read/Write/Edit）使用 paramiko 的 SFTP 功能。命令执行（Bash/Glob/Grep）使用持久 bash session。

理由：SFTP 是二进制安全协议，文件内容传输不经过 shell，不存在特殊字符转义问题。持久 bash session 保证跨工具调用的 shell 状态（cwd、环境变量）持久有效。

### 3.6 持久 bash session 使用 Sentinel 协议

在持久 bash 进程的 stdin/stdout 管道上，用随机 UUID 作为命令结束标记，判断一次命令调用何时完成。协议细节见 4.3 节。

理由：持久 bash 的 stdout 是持续开放的流，没有内建的"命令结束"信号。Sentinel 协议是在不关闭 channel 的前提下确定命令边界的标准方法，已在多个生产系统中验证。

### 4.2 connection.py — SSH 连接管理

**职责**：封装 paramiko SSH 连接，提供两种执行接口：`exec()`（无状态单次执行）和 `get_bash_session()`（持久 shell session 访问）。同一连接上复用 SFTP session。

**核心类**：

```python
# Fragment
@dataclass
class HostConfig:
    hostname: str
    user: str
    port: int = 22
    key_path: Optional[str] = None   # None 则使用 ssh-agent 或默认 key
    password: Optional[str] = None
    jump_host: Optional[str] = None  # 引用 config.yaml 中另一个 host 名称
    connect_timeout: float = 10.0
    keepalive_interval: int = 30     # SSH keepalive 间隔（秒），0 表示禁用

@dataclass
class ExecResult:
    stdout: str
    stderr: str
    exit_code: int

class SSHConnection:
    def connect(self) -> None:
        """
        建立 SSH 连接。jump_host 不为 None 时通过跳板机建立隧道。
        连接建立后立即启用 SSH 层 keepalive：
            transport.set_keepalive(config.keepalive_interval)
        keepalive 会定期向服务器发送 SSH 协议层心跳包，产生少量流量，
        防止 VPN 或防火墙因空闲超时切断连接。
        keepalive_interval 默认 30 秒，应小于 VPN 的空闲超时阈值。
        """

    def exec(self, command: str, timeout: float = 30.0) -> ExecResult:
        """
        无状态单次执行。每次调用创建新 channel，执行完毕后关闭。
        供 Glob、Grep 工具使用。
        """

    def get_sftp(self) -> paramiko.SFTPClient:
        """返回 SFTP session（懒初始化，复用同一实例）。供 Read/Write/Edit 使用。"""

    def get_bash_session(self) -> BashSession:
        """返回持久 bash session（懒初始化，整个连接生命周期内唯一）。供 Bash 使用。"""

    def close(self) -> None:
        """关闭所有 channel、SFTP session 和 SSH 连接。"""
```

**ProxyJump 实现**：

```python
# Fragment — 跳板机连接流程
# 1. 先连接跳板机
jump_client = paramiko.SSHClient()
jump_client.connect(jump_config.hostname, username=jump_config.user, ...)

# 2. 通过跳板机建立到目标的 TCP 隧道 channel
tunnel = jump_client.get_transport().open_channel(
    "direct-tcpip",
    (target_config.hostname, target_config.port),
    ("localhost", 0)
)

# 3. 在隧道上建立目标 SSH 连接
target_client = paramiko.SSHClient()
target_client.connect(target_config.hostname, sock=tunnel, ...)
```

### 4.3 bash_session.py — 持久 Shell Session

**职责**：在远程维护一个长期存活的 bash 进程，跨调用保持 shell 状态（cwd、环境变量）。

**Sentinel 协议**：

持久 bash 的 stdout 是连续流，无法通过关闭 channel 判断命令边界。协议如下：

```
发送到 bash stdin:
    {user_command}
    echo "RMCP_SENTINEL_{uuid}_EXIT_$?"

从 bash stdout 读取:
    行 1..N  → 命令输出，收集到 output
    行 N+1   → RMCP_SENTINEL_{uuid}_EXIT_{exit_code}
               读到此行即停止，提取 exit_code

uuid 每次 execute() 调用独立生成（uuid4().hex），防止命令输出意外包含 sentinel。
sentinel 行本身不包含在返回给调用方的 output 中。
```

**核心类**：

```python
# Fragment
class BashResult:
    output: str      # stdout + stderr 合并（bash 进程以 2>&1 启动）
    exit_code: int

class BashSession:
    def __init__(self, transport: paramiko.Transport):
        self._channel: paramiko.Channel
        self._output_queue: queue.Queue  # 后台线程写入，execute() 消费

    def start(self) -> None:
        """
        在 SSH channel 上启动持久 bash 进程并初始化：
            bash --norc --noprofile
        初始化序列（启动后立即执行）：
            set +m            # 禁用 job control 通知
            set +o histexpand # 禁用 ! 历史扩展
            export PS1=''     # 清空提示符
            export TERM=dumb  # 禁用终端控制序列
            exec 2>&1         # 合并 stderr 到 stdout
        """

    def execute(self, command: str, timeout: float = 60.0) -> BashResult:
        """
        执行命令，返回输出和退出码。
        超时时向 bash 发送 Ctrl-C（b'\\x03'），抛出 TimeoutError。
        bash 进程本身不受影响，下次调用仍可使用。
        """

    def _reader_thread(self) -> None:
        """
        后台线程，持续从 channel recv() 读取数据，
        按行分割后放入 _output_queue。
        必须持续读取——paramiko channel 缓冲区有限，
        不及时消费会导致远程 bash 写满缓冲区后阻塞（死锁）。
        """
```

**初始化序列说明**：

`set +m` 禁用 job control 消息（否则后台命令完成时 bash 会输出 `[1]+ Done ...`，污染 sentinel 解析）。`export PS1=''` 清空提示符，否则每条命令前都有提示符字符串混入输出。这两项是 sentinel 协议正确工作的前提。

### 4.4 tools/read.py

**实现**：通过 SFTP 读取远程文件，在本地截取指定行范围后返回。

```python
# Fragment
def read(sftp: paramiko.SFTPClient, file_path: str,
         offset: int = 1, limit: int = 2000) -> str:
    try:
        with sftp.file(file_path, 'r') as f:
            lines = f.read().decode('utf-8').splitlines(keepends=True)
    except FileNotFoundError:
        return f"Error: File not found: {file_path}"

    # offset 是 1-based 行号
    start = max(0, offset - 1)
    selected = lines[start : start + limit]

    # 添加行号前缀，格式与 Claude Code 内置 Read 一致
    result = ""
    for i, line in enumerate(selected, start=start+1):
        result += f"     {i}\t{line}"
    return result
```

**注意**：整个文件通过 SFTP 传输到本地再截取。对于极大文件（>100 MB）可改为 `exec("sed -n '{start},{end}p' {file}")` 在远程截取，仅传输指定行。当前版本不做此优化，实施者可在验收后按需添加。

### 4.5 tools/write.py

**实现**：通过 SFTP 直接写入，绕过 shell，二进制安全。

```python
# Fragment
def write(sftp: paramiko.SFTPClient, conn: SSHConnection,
          file_path: str, content: str) -> str:
    # 确保父目录存在
    parent = str(pathlib.PurePosixPath(file_path).parent)
    conn.exec(f"mkdir -p {shlex.quote(parent)}")

    # SFTP 写入，内容不经过 shell，含任意特殊字符均安全
    with sftp.file(file_path, 'w') as f:
        f.write(content.encode('utf-8'))

    return f"Successfully wrote {len(content)} characters to {file_path}"
```

### 4.6 tools/edit.py

**实现**：通过 SFTP 读取全文，在本地 Python 中做字符串替换后写回。

```python
# Fragment
def edit(sftp: paramiko.SFTPClient, file_path: str,
         old_string: str, new_string: str) -> str:
    try:
        with sftp.file(file_path, 'r') as f:
            content = f.read().decode('utf-8')
    except FileNotFoundError:
        return f"Error: File not found: {file_path}"

    count = content.count(old_string)
    if count == 0:
        return f"Error: old_string not found in {file_path}"
    if count > 1:
        return (f"Error: old_string found {count} times in {file_path}. "
                f"Provide more context to match uniquely.")

    new_content = content.replace(old_string, new_string, 1)

    with sftp.file(file_path, 'w') as f:
        f.write(new_content.encode('utf-8'))

    return f"Successfully edited {file_path}"
```

Edit 每次调用涉及一次完整文件读取和写回（SFTP）。对于源代码和配置文件（通常 < 1 MB）开销可以接受。

### 4.7 tools/bash.py

**实现**：调用 BashSession.execute()，透传结果。

```python
# Fragment
def bash(session: BashSession, command: str,
         description: str = "", timeout: float = 60.0) -> str:
    try:
        result = session.execute(command, timeout=timeout)
    except TimeoutError:
        return f"Error: Command timed out after {timeout}s"

    output = result.output
    if result.exit_code != 0:
        output += f"\n[Exit code: {result.exit_code}]"
    return output
```

### 4.8 tools/glob.py

**实现**：在远程执行 `find` 命令，只传回文件路径列表。

```python
# Fragment
def glob(conn: SSHConnection, pattern: str, path: str = ".") -> str:
    # 将 glob pattern 转换为 find 可用的形式
    # "**/*.py" → find path -name "*.py"（提取文件名部分）
    # "src/*.c" → find path/src -maxdepth 1 -name "*.c"
    name_part = pathlib.PurePosixPath(pattern).name
    cmd = f"find {shlex.quote(path)} -name {shlex.quote(name_part)} | sort"

    result = conn.exec(cmd)
    if result.exit_code not in (0, 1):
        return f"Error: {result.stderr}"
    if not result.stdout.strip():
        return "No files found matching pattern"
    return result.stdout
```

**注意**：当前实现只使用 `find -name`，对于含路径层级的 pattern（如 `src/**/*.py`）只匹配文件名部分。实施者需在验收测试中确认此行为是否满足 Claude Code 的使用需求，必要时完善 pattern 转换逻辑。

### 4.9 tools/grep.py

**实现**：在远程执行 `grep`，只传回匹配行。

```python
# Fragment
def grep(conn: SSHConnection, pattern: str, path: str,
         include: str = "", case_insensitive: bool = False) -> str:
    flags = "-rn"
    if case_insensitive:
        flags += "i"
    include_opt = f"--include={shlex.quote(include)}" if include else ""

    cmd = (f"grep {flags} {include_opt} "
           f"-E {shlex.quote(pattern)} {shlex.quote(path)} "
           f"| head -200")

    result = conn.exec(cmd)
    # grep 返回码：0=有匹配，1=无匹配，2=错误
    if result.exit_code == 2:
        return f"Error: {result.stderr}"
    if result.exit_code == 1:
        return "No matches found"
    return result.stdout
```

`head -200` 防止匹配结果过多时传输量过大。200 行是默认上限，实施者可通过配置参数暴露此值。

### 4.10 server.py — MCP Server 主框架

**职责**：初始化 SSH 连接，注册所有工具，启动 MCP stdio server 主循环。

```python
# Fragment
from mcp.server import Server
from mcp.server.stdio import stdio_server

app = Server("remote-mcp")

@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(name="Read",  description="Read a file on the remote server.",
             inputSchema=READ_SCHEMA),
        Tool(name="Write", description="Write content to a file on the remote server.",
             inputSchema=WRITE_SCHEMA),
        Tool(name="Edit",  description="Edit a file by replacing an exact string.",
             inputSchema=EDIT_SCHEMA),
        Tool(name="Bash",  description="Execute a shell command on the remote server. "
                                        "Shell state persists across calls.",
             inputSchema=BASH_SCHEMA),
        Tool(name="Glob",  description="Find files matching a glob pattern.",
             inputSchema=GLOB_SCHEMA),
        Tool(name="Grep",  description="Search file contents for a pattern.",
             inputSchema=GREP_SCHEMA),
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    result = dispatch(name, arguments, conn)
    return [TextContent(type="text", text=result)]

async def main(host_name: str, config_path: str):
    config = load_config(config_path)
    conn = SSHConnection(config.hosts[host_name])
    conn.connect()
    try:
        async with stdio_server() as (read_stream, write_stream):
            await app.run(read_stream, write_stream,
                          app.create_initialization_options())
    finally:
        conn.close()
```

## 7. 错误处理策略

### 7.1 错误返回格式

所有工具在失败时返回以 `Error:` 开头的字符串，而不是抛出异常。Claude Code 读到错误信息后可据此调整策略。

```
Error: File not found: /etc/nonexistent.conf
Error: old_string not found in /opt/app/config.py
Error: old_string found 3 times in /opt/app/config.py. Provide more context to match uniquely.
Error: Command timed out after 60 seconds
Error: SSH connection lost. Reconnect failed: Connection refused
```

### 7.2 错误分类

| 错误 | 触发场景 | 处理 |
|------|----------|------|
| 连接失败 | 主机不可达、认证失败 | server 启动时报错退出，不进入主循环 |
| 连接中断 | 网络抖动、VPN 超时 | 自动重连一次；失败则返回 Error 字符串 |
| 命令超时 | 长时间运行命令 | 发送 Ctrl-C，返回已收到输出 + Error 提示，session 保持 |
| 文件不存在 | Read/Edit 目标不存在 | 返回 Error 字符串 |
| old_string 不唯一 | Edit 匹配失败或多次匹配 | 返回 Error 字符串说明原因 |
| bash session 崩溃 | 远程 bash 进程意外退出、连接重建后 | 下次 Bash 调用时自动重建 session；**重建后 shell 状态（cwd、环境变量）全部丢失**，返回的 Error 字符串中需说明此情况 |

### 7.3 重连与状态丢失通知

**问题**：连接中断后自动重连，bash session 随之重建，shell 状态（cwd、环境变量、已 source 的文件）全部丢失。如果 agent 不知道重连发生，它会继续基于旧状态假设操作——例如重连前执行了 `cd /opt/project`，重连后 agent 继续用相对路径，但 cwd 已回到 `$HOME`，导致命令作用在错误目录。

**设计原则**：重连发生时，不静默恢复，必须让 agent 感知到状态已重置。

**实现**：`SSHConnection` 维护一个 `_reconnected` 标志。重连成功后将其置为 `True`。每个工具在执行前检查此标志，若为 `True`，则在本次工具返回结果中**前置一条警告**，同时将标志清除（只警告一次）。

```python
# Fragment
class SSHConnection:
    def __init__(self, ...):
        self._reconnected: bool = False   # 重连后置 True，工具读取后清除

    def _do_reconnect(self) -> None:
        """重连成功后调用。"""
        self.connect()
        self._reconnected = True
        # bash session 已失效，下次 get_bash_session() 会重建

    def check_and_clear_reconnect_flag(self) -> bool:
        """工具调用开始时检查。返回 True 表示刚刚发生过重连。"""
        flag = self._reconnected
        self._reconnected = False
        return flag
```

每个工具在返回结果前调用此检查：

```python
# Fragment — 所有工具共用的前置检查，在 server.py 的 call_tool() 中统一处理
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    prefix = ""
    if conn.check_and_clear_reconnect_flag():
        prefix = (
            "[WARNING] SSH connection was lost and has been re-established. "
            "The remote bash session has been reset: working directory is now $HOME, "
            "all environment variables set in previous commands are lost. "
            "Use absolute paths and re-run any necessary setup commands.\n\n"
        )
    result = dispatch(name, arguments, conn)
    return [TextContent(type="text", text=prefix + result)]
```

**警告文本的设计要求**：

- 明确说明**发生了什么**（连接重建）
- 明确说明**状态变化**（cwd 回到 $HOME，环境变量丢失）
- 给出**具体的应对指示**（使用绝对路径，重新执行初始化命令）

这三点缺一不可。模糊的"连接已恢复"对 agent 没有指导意义。

**重连失败的处理**：

重连失败时不警告，直接返回 `Error: SSH connection lost and reconnect failed`。此时 agent 无法继续操作，应由用户介入检查网络。

```python
# Fragment
def exec_with_retry(self, command: str) -> ExecResult:
    try:
        return self._exec(command)
    except (paramiko.SSHException, EOFError, OSError):
        try:
            self._do_reconnect()
            return self._exec(command)
        except Exception as e:
            raise ConnectionError(f"SSH reconnect failed: {e}")
```

## 8. 实施计划

按依赖关系从底层向上实施，每阶段有独立的验收标准。

### 阶段 1：connection.py

验收标准：

- `conn.exec("echo hello")` 返回 `stdout="hello\n"`, `exit_code=0`
- `conn.exec("cat /nonexistent")` 返回非零 `exit_code`
- `conn.get_sftp()` 返回可用的 `SFTPClient`
- jump_host 配置下连接成功
- 连接建立后 `transport.get_keepalive()` 返回配置值（验证 keepalive 已启用）
- 模拟断连后调用任意工具，返回结果包含 WARNING 前缀，且下次调用不再包含

### 阶段 2：bash_session.py

此阶段最容易出错，建议用独立测试脚本充分验证再集成。

验收标准：

- `session.execute("cd /tmp && pwd")` 返回 `/tmp`
- 紧接着 `session.execute("pwd")` 仍返回 `/tmp`（cwd 持久）
- `session.execute("export FOO=bar")` 后，`session.execute("echo $FOO")` 返回 `bar`（env 持久）
- `session.execute("echo 'it'\\''s a $test'")` 返回原始字符串不损坏（特殊字符）
- `session.execute("sleep 100", timeout=2)` 在约 2 秒后返回 TimeoutError，session 仍可继续使用

### 阶段 3：Read / Write / Edit

验收标准：

- Write 写入含单引号、双引号、反斜杠、`$VAR`、换行符的内容，Read 读回完全一致
- Edit 替换唯一匹配，返回成功
- Edit 在 old_string 不存在时返回 `Error: old_string not found`
- Edit 在 old_string 出现 2 次时返回包含次数的错误信息
- Read 使用 offset=10, limit=5 只返回第 10-14 行

### 阶段 4：Glob / Grep

验收标准：

- Glob `"*.py"` 在含 Python 文件的目录下返回正确列表
- Grep 在远程 1 GB 文件中搜索关键词，响应时间明显少于传输整个文件的时间
- Grep 对不存在的路径返回 Error 字符串

### 阶段 5：server.py + __main__.py

验收标准：

- `python -m remote_mcp --host prod` 启动后保持运行
- `claude mcp add` 注册后，Claude Code 工具列表出现 6 个工具
- 在 Claude Code 中调用 Bash 执行 `ls -la /tmp`，返回远程目录列表
- 在 Claude Code 中调用 Read 读取远程文件，内容正确

### 阶段 6：配置 + 打包 + README

- `pip install remote-mcp` 后可直接使用
- README 包含完整安装、配置、注册步骤和常见问题

## 9. 已知局限

**交互式命令不支持**：`vim`、`top`、Python REPL 等需要 TTY 的程序无法通过 Bash 工具使用。可通过 `Bash("tmux send-keys -t session 'cmd' Enter")` 间接操作，但不在本版本范围内。

**二进制文件**：Write 和 Edit 只支持 UTF-8 文本。二进制文件传输需单独实现。

**Glob 的 `**` 语义**：当前实现用 `find -name` 只匹配文件名部分，不完全等价于 Python glob 的 `**` 递归路径匹配。在路径层级匹配场景下可能有差异，实施者需在验收测试中确认并决定是否完善。

**Edit 的并发安全**：Edit 的 read-modify-write 不是原子操作。单 agent 顺序执行无问题，多 agent 并发写同一文件可能产生竞争，当前版本不处理此场景。

**大文件 Read 性能**：Read 当前实现通过 SFTP 传输全文再截取，对 > 100 MB 的文件效率低。改进方法是用 `conn.exec("sed -n '{start},{end}p' {file}")` 在远程截取后传输，实施者可在基础功能验收后按需添加。
