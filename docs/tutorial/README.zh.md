# 教程

> English version: [`README.md`](./README.md)

教程是**课程**。它们通过引导初学者完成一段完整的小体验来传授知识。你按照步骤操作，就能看到与文档描述相同的结果。

教程*不是*参考手册（它不逐一列举参数），也不是操作指南（它不预设你已经知道目标是什么——而是带你亲历一个具体目标）。

## 可用教程

| 教程 | 时长 | 前置条件 |
|------|------|----------|
| [第一次远程会话](./first-remote-session.md) | 约 15 分钟 | 一台可以 `ssh` 登录的 Linux 主机；本地安装 Python 3.8+；已安装 Claude Code |

如果你从未使用过 remote-mcp，**从这里开始**。

## 教程的写作规范（面向贡献者）

- 始终以一个成功且可见的结果收尾（"你现在应该看到……"）。
- 每条命令之后都要展示预期输出，让读者能在每一步确认结果。
- 不要解释*为什么*——那是 `explanation/` 的职责。不要列举所有选项——那是 `reference/` 的职责。
- 教程应该是安全可跟随的：不能有损坏用户环境的风险。
- 一篇教程 = 一段旅程。不要添加"备选方案"或"你也可以试试 X"——这些内容属于操作指南。
